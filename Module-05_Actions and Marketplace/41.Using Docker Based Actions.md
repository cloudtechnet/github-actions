## GitHub Actions: 41. Using Docker Based Actions

**Docker based actions** are GitHub Actions that run inside a **Docker container**.

It means GitHub will start a Docker container and run the action code inside that container.

## Why use Docker Based Actions?

Docker based actions are useful when:

You need a fixed environment.
You need specific tools installed.
You want the same result every time.
You want to package your script with dependencies.
You want to avoid installing tools again and again.

## Types of GitHub Actions

There are mainly 3 types:

1. **JavaScript Actions**
2. **Docker Container Actions**
3. **Composite Actions**

Here we are learning **Docker Container Actions**.

## Basic Structure

A Docker based action usually contains:

```text
action.yml
Dockerfile
script file
```

Example folder structure:

```text
my-docker-action/
│
├── action.yml
├── Dockerfile
└── entrypoint.sh
```

## Example: Create Docker Based Action

### Step 1: Create `action.yml`

```yaml
name: Docker Greeting Action
description: A simple Docker based GitHub Action

inputs:
  name:
    description: Name of the person
    required: true
    default: Student

runs:
  using: docker
  image: Dockerfile
```

## Step 2: Create `Dockerfile`

```dockerfile
FROM alpine:latest

COPY entrypoint.sh /entrypoint.sh

RUN chmod +x /entrypoint.sh

ENTRYPOINT ["/entrypoint.sh"]
```

## Step 3: Create `entrypoint.sh`

```bash
#!/bin/sh

echo "Hello $1"
echo "Welcome to GitHub Actions Docker Based Action"
```

## Step 4: Use this action in workflow

Create:

```text
.github/workflows/docker-action.yml
```

```yaml
name: Use Docker Based Action

on:
  push:
    branches: [ main ]

jobs:
  docker-action-job:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Run Custom Docker Action
        uses: ./my-docker-action
        with:
          name: Priya
```

## How it works

When code is pushed to `main`:

1. GitHub starts Ubuntu runner.
2. Workflow checks out repository code.
3. GitHub reads `action.yml`.
4. It builds Docker image using `Dockerfile`.
5. It runs `entrypoint.sh` inside the container.
6. Output will show greeting message.

## Important Note

In Docker actions, inputs are passed as arguments or environment variables.

Better script example:

```bash
#!/bin/sh

echo "Hello ${INPUT_NAME}"
echo "This is running inside Docker container"
```

GitHub automatically converts input `name` into environment variable:

```text
INPUT_NAME
```

## Real-Time Example: Check Application Version

### `entrypoint.sh`

```bash
#!/bin/sh

echo "Checking application version..."
java -version
mvn -version
```

### `Dockerfile`

```dockerfile
FROM maven:3.9-eclipse-temurin-17

COPY entrypoint.sh /entrypoint.sh
RUN chmod +x /entrypoint.sh

ENTRYPOINT ["/entrypoint.sh"]
```

This Docker action gives a ready environment with **Java 17 + Maven**.

## Advantages

Same environment for every run.
Easy to manage dependencies.
Useful for custom tools.
Good for DevOps automation.
Works well for build, scan, test, and deploy tasks.

## Limitations

Docker based actions run only on Linux runners.
They may take more time because Docker image must be built or pulled.
Not suitable for very small simple tasks.

## Simple Definition

**Docker based actions are GitHub Actions that run inside a Docker container, giving a fixed and reusable environment for automation tasks.**

## 35. Using Predefined Marketplace Actions

**Predefined Marketplace Actions** means ready-made actions available in **GitHub Marketplace**. We can directly use them in our workflow instead of writing everything from scratch.

## What are Predefined Actions?

They are reusable automation blocks.

Examples:

```yaml
uses: actions/checkout@v4
```

```yaml
uses: actions/setup-java@v4
```

```yaml
uses: docker/build-push-action@v6
```

## Why we use them

They save time.
They reduce manual scripting.
They are easy to reuse.
They help in build, test, deployment, Docker, cloud, and notification tasks.

## Syntax

```yaml
- name: Step name
  uses: action-name@version
```

Example:

```yaml
- name: Checkout Code
  uses: actions/checkout@v4
```

Here:

`uses` means we are using an existing action.
`actions/checkout` is the action name.
`@v4` is the version.

## Example: Java Maven Project

Create file:

```text
.github/workflows/java-ci.yml
```

```yaml
name: Java CI Using Marketplace Actions

on:
  push:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Repository
        uses: actions/checkout@v4

      - name: Setup Java
        uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: 17

      - name: Build Project
        run: mvn clean install
```

## Explanation

When code is pushed to `main` branch:

1. Workflow starts automatically.
2. `actions/checkout@v4` downloads repository code.
3. `actions/setup-java@v4` installs Java 17.
4. Maven command builds the project.

## Example: Node.js Project

```yaml
name: Node.js CI

on:
  push:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Install Dependencies
        run: npm install

      - name: Run Tests
        run: npm test
```

## Common Marketplace Actions

`actions/checkout` → checkout repository code
`actions/setup-java` → install Java
`actions/setup-node` → install Node.js
`docker/login-action` → login to Docker Hub
`docker/build-push-action` → build and push Docker image
`azure/webapps-deploy` → deploy app to Azure
`slackapi/slack-github-action` → send Slack message

## Best Practices

Use trusted actions.
Use fixed versions like `@v4`, not random branches.
Read action documentation.
Check required inputs.
Do not expose secrets in workflow logs.
Use GitHub Secrets for passwords and tokens.

## Simple Definition

**Using predefined Marketplace Actions means using ready-made GitHub Actions from Marketplace to automate tasks like checkout, setup, build, test, Docker, deploy, and notifications.**

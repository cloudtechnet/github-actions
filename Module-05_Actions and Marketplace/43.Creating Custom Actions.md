## 43. Creating Custom Actions

**Custom Actions** means creating our own reusable GitHub Action for automation.

Instead of writing the same steps again and again in every workflow, we can create one custom action and reuse it.

## Why create Custom Actions?

Custom actions are useful when:

You have repeated tasks.
You want reusable automation.
You want standard process for all projects.
You want to reduce workflow file size.
You want to create company-specific CI/CD steps.

## Types of Custom Actions

1. **Composite Action**
   Uses multiple shell commands together.

2. **JavaScript Action**
   Written using Node.js.

3. **Docker Action**
   Runs inside Docker container.

## Example: Create a Composite Custom Action

### Folder Structure

```text
.github/actions/greeting-action/
│
└── action.yml
```

## Step 1: Create `action.yml`

```yaml
name: Greeting Custom Action
description: This custom action prints welcome message

inputs:
  student-name:
    description: Name of the student
    required: true
    default: Student

runs:
  using: composite
  steps:
    - name: Print Welcome Message
      shell: bash
      run: |
        echo "Hello ${{ inputs.student-name }}"
        echo "Welcome to GitHub Actions Custom Action"
        echo "This is a reusable custom action"
```

## Step 2: Use Custom Action in Workflow

Create workflow file:

```text
.github/workflows/custom-action.yml
```

```yaml
name: Use Custom Action

on:
  push:
    branches: [ main ]

jobs:
  run-custom-action:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Run Greeting Custom Action
        uses: ./.github/actions/greeting-action
        with:
          student-name: Priya
```

## How it works

When code is pushed to `main`:

1. Workflow starts.
2. Code is downloaded using `actions/checkout`.
3. GitHub reads the custom action from `.github/actions/greeting-action`.
4. It executes the commands inside `action.yml`.
5. Output will show welcome messages.

## Real-Time Example: Custom Maven Build Action

### Folder Structure

```text
.github/actions/maven-build/
└── action.yml
```

### `action.yml`

```yaml
name: Maven Build Custom Action
description: Reusable Maven build action

inputs:
  java-version:
    description: Java version
    required: true
    default: '17'

runs:
  using: composite
  steps:
    - name: Setup Java
      uses: actions/setup-java@v4
      with:
        distribution: temurin
        java-version: ${{ inputs.java-version }}

    - name: Build Maven Project
      shell: bash
      run: mvn clean install
```

### Workflow

```yaml
name: Java Build Using Custom Action

on:
  push:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Run Maven Build Action
        uses: ./.github/actions/maven-build
        with:
          java-version: '17'
```

## Explanation

This custom action:

Sets up Java.
Runs Maven build.
Can be reused in multiple Java projects.
Keeps workflow clean and professional.

## Best Practices

Keep action name clear.
Use inputs for reusable values.
Add proper description.
Use versioning if action is in separate repository.
Do not hardcode secrets.
Test custom action before using in production.

## Simple Definition

**Creating custom actions means building our own reusable GitHub Action to automate repeated tasks like build, test, scan, deploy, or notifications.**

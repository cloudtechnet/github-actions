## GitHub Actions: 48. Publishing Custom Actions

**Publishing Custom Actions** means making your own GitHub Action available so other repositories or users can use it.

You can publish actions in:

GitHub repository
GitHub Marketplace
Organization shared repository

## Why publish custom actions?

✅ Reuse in many projects
✅ Share with team members
✅ Standardize CI/CD process
✅ Save development time
✅ Help other developers automate tasks

## Types of actions we can publish

1. **Composite Action**
2. **JavaScript Action**
3. **Docker Action**

## Example: Publish Composite Action

### Step 1: Create a new repository

Example repository name:

```text
java-maven-build-action
```

## Step 2: Create `action.yml`

```yaml
name: Java Maven Build Action
description: Setup Java and build Maven project

inputs:
  java-version:
    description: Java version to use
    required: true
    default: '17'

runs:
  using: composite
  steps:
    - name: Setup Java
      uses: actions/setup-java@v4
      with:
        distribution: temurin
        java-version: ${{ inputs.java-version }}

    - name: Build Maven Project
      shell: bash
      run: mvn clean install
```

## Step 3: Add README file

````markdown
# Java Maven Build Action

This action sets up Java and builds a Maven project.

## Usage

```yaml
- name: Java Build
  uses: username/java-maven-build-action@v1
  with:
    java-version: '17'
````

````

## Step 4: Push code to GitHub

```bash
git add .
git commit -m "Create Java Maven build action"
git push origin main
````

## Step 5: Create version tag

```bash
git tag v1
git push origin v1
```

## Step 6: Use published action in another repository

```yaml
name: Use Published Custom Action

on:
  push:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Use Java Maven Build Action
        uses: username/java-maven-build-action@v1
        with:
          java-version: '17'
```

## Explanation

```yaml
uses: username/java-maven-build-action@v1
```

Means:

`username` → GitHub account or organization
`java-maven-build-action` → action repository name
`v1` → version tag

## Publishing to GitHub Marketplace

To publish in GitHub Marketplace:

1. Repository must be public.
2. Action must have `action.yml`.
3. README should explain usage clearly.
4. Create a release.
5. Choose **Publish this Action to GitHub Marketplace**.
6. Add category, logo, and description.

## Best Practices

Use clear action name.
Write proper README.
Add inputs and outputs.
Use version tags like `v1`, `v2`.
Do not hardcode secrets.
Test action before publishing.
Keep action focused on one task.
Add examples for users.

## Simple Definition

**Publishing Custom Actions means uploading your own action to GitHub and versioning it so others can use it in their workflows using `uses:`.**

## GitHub Actions: 50. Multi OS Testing using Matrix

**Multi OS Testing using Matrix** means running the same workflow on multiple operating systems automatically.

Example:

✅ Ubuntu
✅ Windows
✅ macOS

This helps us check whether our application works correctly on different platforms.

## Why Multi OS Testing is useful

Some projects work differently on different operating systems.

For example:

Linux command:

```bash
ls
```

Windows command:

```powershell
dir
```

Path format can also be different:

```text
Linux/macOS: /home/project/file.txt
Windows: C:\project\file.txt
```

So testing on multiple OS is important.

## Basic Syntax

```yaml
strategy:
  matrix:
    os: [ubuntu-latest, windows-latest, macos-latest]
```

Then use it here:

```yaml
runs-on: ${{ matrix.os }}
```

## Example: Java Maven Multi OS Testing

Create file:

```text
.github/workflows/multi-os-test.yml
```

```yaml
name: Multi OS Java Testing

on:
  push:
    branches: [ main ]

jobs:
  test:
    runs-on: ${{ matrix.os }}

    strategy:
      fail-fast: false
      matrix:
        os: [ubuntu-latest, windows-latest, macos-latest]
        java-version: [17]

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Setup Java
        uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: ${{ matrix.java-version }}

      - name: Run Maven Tests
        run: mvn test
```

## Explanation

This workflow runs the same test job on:

```text
ubuntu-latest
windows-latest
macos-latest
```

So total 3 jobs will run.

```text
Ubuntu + Java 17
Windows + Java 17
macOS + Java 17
```

## Example: Multi OS + Multiple Java Versions

```yaml
name: Multi OS and Java Version Testing

on: push

jobs:
  test:
    runs-on: ${{ matrix.os }}

    strategy:
      fail-fast: false
      matrix:
        os: [ubuntu-latest, windows-latest, macos-latest]
        java-version: [17, 21]

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Setup Java
        uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: ${{ matrix.java-version }}

      - name: Run Tests
        run: mvn test
```

## How many jobs will run?

```text
3 OS × 2 Java versions = 6 jobs
```

Jobs:

```text
Ubuntu + Java 17
Ubuntu + Java 21
Windows + Java 17
Windows + Java 21
macOS + Java 17
macOS + Java 21
```

## Node.js Multi OS Example

```yaml
name: Node.js Multi OS Testing

on: push

jobs:
  test:
    runs-on: ${{ matrix.os }}

    strategy:
      matrix:
        os: [ubuntu-latest, windows-latest, macos-latest]
        node-version: [18, 20]

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}

      - name: Install Dependencies
        run: npm install

      - name: Run Tests
        run: npm test
```

## Using OS-specific commands

Sometimes command is different for Windows and Linux/macOS.

```yaml
- name: Linux/macOS Command
  if: runner.os != 'Windows'
  run: ls -la

- name: Windows Command
  if: runner.os == 'Windows'
  run: dir
```

## Important Matrix Options

### `fail-fast: false`

```yaml
strategy:
  fail-fast: false
```

This means if one OS job fails, other OS jobs will continue running.

### `max-parallel`

```yaml
strategy:
  max-parallel: 2
```

This runs only 2 matrix jobs at the same time.

## Real-Time Use Case

Suppose we are building a Java application used by different developers.

Some developers use Windows.
Some use Ubuntu.
Some use macOS.

So we test on all 3 operating systems before merging code.

## Best Practices

Use matrix for OS testing.
Keep OS-specific commands separate.
Use `fail-fast: false` for better results.
Avoid hardcoded file paths.
Test scripts on all OS platforms.
Use versioned actions like `actions/checkout@v4`.

## Simple Definition

**Multi OS Testing using Matrix means testing the same project automatically on Ubuntu, Windows, and macOS using one GitHub Actions workflow.**

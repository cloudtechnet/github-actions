## 54. Debugging GitHub Workflows

**Debugging GitHub Workflows** means finding and fixing errors in GitHub Actions workflow runs.

Errors can happen because of:

Wrong YAML syntax
Wrong indentation
Missing secrets
Wrong file path
Wrong command
Failed build or test
Incorrect action version

## Common Debugging Steps

### 1. Check Actions Logs

Go to:

```text
GitHub Repository → Actions → Select Workflow Run → Click Failed Job
```

There you can see which step failed.

## Example Failed Workflow

```yaml
name: Debug Example

on: push

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Build Project
        run: mvn clean install
```

If Maven is not installed or `pom.xml` is missing, the build step fails.

## 2. Add Debug Commands

```yaml
- name: Debug Current Directory
  run: pwd

- name: List Files
  run: ls -la
```

These commands help confirm where the workflow is running and what files are available.

## 3. Print Environment Variables

```yaml
- name: Print GitHub Info
  run: |
    echo "Branch: ${{ github.ref }}"
    echo "Commit: ${{ github.sha }}"
    echo "Runner OS: ${{ runner.os }}"
```

## 4. Enable Step Debug Logs

Add repository secret:

```text
ACTIONS_STEP_DEBUG = true
```

Then rerun workflow.

This gives more detailed logs for each step.

## 5. Use Continue on Error for Testing

```yaml
- name: Run Tests
  run: mvn test
  continue-on-error: true
```

This allows the workflow to continue even if this step fails.

## 6. Debug Secrets Safely

Never print secrets directly.

Wrong:

```yaml
run: echo ${{ secrets.DEPLOY_TOKEN }}
```

Correct:

```yaml
run: |
  if [ -z "${{ secrets.DEPLOY_TOKEN }}" ]; then
    echo "Secret is missing"
  else
    echo "Secret is available"
  fi
```

## Real-Time Example: Java Workflow Debugging

```yaml
name: Java Debug Workflow

on: push

jobs:
  debug-build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Show Current Folder
        run: pwd

      - name: Show Project Files
        run: ls -la

      - name: Setup Java
        uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: 17

      - name: Check Java Version
        run: java -version

      - name: Check Maven Version
        run: mvn -version

      - name: Build Project
        run: mvn clean install
```

## Explanation

This workflow helps debug by checking:

Current folder
Available files
Java version
Maven version
Build command output

## Common Errors and Fixes

| Error               | Reason                | Fix                               |
| ------------------- | --------------------- | --------------------------------- |
| `yaml: line error`  | Wrong indentation     | Fix spaces                        |
| `pom.xml not found` | Wrong directory       | Use `cd folder-name`              |
| `Permission denied` | Script not executable | Use `chmod +x script.sh`          |
| `Secret not found`  | Secret not added      | Add secret in repository settings |
| `command not found` | Tool missing          | Install or setup tool             |

## Best Practices

Read logs from top to bottom.
Check the exact failed step.
Add `pwd` and `ls -la` for path issues.
Use setup actions for required tools.
Do not print secrets.
Rerun failed jobs after fixing.

## Simple Definition

**Debugging GitHub Workflows means checking workflow logs, identifying failed steps, and fixing YAML, command, path, secret, or environment issues.**

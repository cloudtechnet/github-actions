## 55. Enable Step Debug Logging

**Step Debug Logging** means enabling extra detailed logs for GitHub Actions steps.

It helps us find the exact reason why a workflow step is failing.

## Why use Step Debug Logging?

It helps to debug:

Wrong commands
Missing variables
Action errors
Tool installation issues
Secret/configuration problems
Build or deployment failures

## How to Enable Step Debug Logging

Go to your repository:

```text
Repository → Settings → Secrets and variables → Actions → New repository secret
```

Create this secret:

```text
Name: ACTIONS_STEP_DEBUG
Value: true
```

After adding this secret, rerun the workflow.

## Example Workflow

```yaml
name: Step Debug Logging Example

on:
  push:
    branches: [ main ]

jobs:
  debug:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Print Debug Info
        run: |
          echo "Current directory:"
          pwd
          echo "Files:"
          ls -la

      - name: Build Project
        run: mvn clean install
```

## Explanation

When `ACTIONS_STEP_DEBUG=true` is added:

GitHub shows more detailed logs for each step.
You can see extra internal action details.
It becomes easier to identify where the issue is happening.

## Important Note

Do not print secrets in logs.

Wrong:

```yaml
run: echo ${{ secrets.DEPLOY_TOKEN }}
```

Correct:

```yaml
run: echo "Secret is configured safely"
```

## Simple Definition

**Enable Step Debug Logging means adding `ACTIONS_STEP_DEBUG=true` in GitHub Secrets to get detailed logs for workflow steps and troubleshoot errors easily.**

## 63. NPM Package Caching

**NPM Package Caching** means saving Node.js package download cache so GitHub Actions can reuse it in future workflow runs.

This helps avoid downloading the same packages again and again.

## Why use NPM cache?

✅ Faster `npm install`
✅ Saves workflow time
✅ Reduces repeated downloads
✅ Improves CI/CD performance

## NPM Cache Path

Common npm cache location:

```text
~/.npm
```

## Example: NPM Package Caching

```yaml
name: NPM Package Cache

on:
  push:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Cache NPM Packages
        uses: actions/cache@v4
        with:
          path: ~/.npm
          key: npm-cache-${{ runner.os }}-${{ hashFiles('package-lock.json') }}
          restore-keys: |
            npm-cache-${{ runner.os }}-

      - name: Install Dependencies
        run: npm ci

      - name: Run Tests
        run: npm test
```

## Explanation

```yaml
path: ~/.npm
```

This tells GitHub what folder to cache.

```yaml
key: npm-cache-${{ runner.os }}-${{ hashFiles('package-lock.json') }}
```

This creates a unique cache key based on operating system and `package-lock.json`.

```yaml
restore-keys:
  npm-cache-${{ runner.os }}-
```

This is a fallback cache if exact key is not found.

## Why use `npm ci`?

```bash
npm ci
```

is recommended for CI/CD because it installs dependencies exactly from `package-lock.json`.

## Easier Method: Built-in NPM Cache

```yaml
name: Node Built-in NPM Cache

on: push

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Setup Node with Cache
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm

      - name: Install Dependencies
        run: npm ci

      - name: Run Tests
        run: npm test
```

## Real-Time Example

Without cache:

```text
Every workflow run downloads npm packages again.
Build takes more time.
```

With cache:

```text
GitHub restores npm package cache.
npm install becomes faster.
```

## Best Practices

Use `npm ci` in GitHub Actions.
Use `package-lock.json` in cache key.
Use OS-based cache keys.
Do not cache `node_modules` unless really needed.
Prefer `actions/setup-node` built-in cache for simple projects.

## Simple Definition

**NPM Package Caching in GitHub Actions means storing npm package cache and reusing it in future workflow runs to make Node.js builds faster.**

## 62. Artifact Retention Policies

**Artifact Retention Policy** means deciding how long uploaded artifacts should be stored in GitHub Actions before they are automatically deleted.

Artifacts can include:

✅ Build files
✅ `.jar` or `.war` files
✅ Test reports
✅ Logs
✅ Coverage reports
✅ Deployment packages

## Why retention policy is important?

Retention policies help to:

✅ Save GitHub storage space
✅ Automatically clean old artifacts
✅ Keep artifacts only when needed
✅ Reduce unnecessary storage usage
✅ Manage CI/CD efficiently

## Default Retention Period

GitHub stores artifacts for a default number of days depending on repository settings.

Common default:

```text
90 days
```

After that, artifacts are deleted automatically.

## Example: Set Artifact Retention Days

```yaml
name: Artifact Retention Example

on:
  push:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Build Application
        run: mvn clean package

      - name: Upload Artifact
        uses: actions/upload-artifact@v4
        with:
          name: java-build
          path: target/*.jar
          retention-days: 7
```

## Explanation

```yaml
retention-days: 7
```

Means artifact will be stored for:

```text
7 days
```

After 7 days, GitHub automatically deletes it.

## Real-Time Example

Suppose a company creates daily build artifacts.

They only need artifacts for:

```text
7 days
```

So old build files are removed automatically.

## Example: Long Retention for Release Builds

```yaml
name: Release Artifact Storage

on:
  release:
    types: [published]

jobs:
  release-build:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Build Release
        run: mvn clean package

      - name: Upload Release Artifact
        uses: actions/upload-artifact@v4
        with:
          name: release-package
          path: target/*.jar
          retention-days: 30
```

## Explanation

Release artifacts are important, so they are stored for:

```text
30 days
```

## Important Notes

Retention days cannot exceed repository or organization limits.

Artifact deletion happens automatically after retention period ends.

## Repository-Level Retention Settings

You can configure default retention here:

```text
Repository → Settings → Actions → Artifact and log retention
```

## Best Practices

Use shorter retention for temporary builds.
Use longer retention for release artifacts.
Do not keep unnecessary artifacts for long time.
Clean old artifacts regularly.
Use meaningful artifact names.

## Simple Definition

**Artifact Retention Policy means controlling how many days GitHub Actions artifacts are stored before automatic deletion.**

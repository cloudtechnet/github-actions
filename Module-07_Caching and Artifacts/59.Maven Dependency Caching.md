## 64. Maven Dependency Caching

**Maven Dependency Caching** means saving Maven downloaded dependencies so GitHub Actions can reuse them in future workflow runs.

Maven dependencies are usually stored in:

```text
~/.m2/repository
```

## Why use Maven caching?

Without cache:

```text
Every workflow downloads dependencies again.
Build takes more time.
```

With cache:

```text
Dependencies are restored from cache.
Build becomes faster.
```

## Easy Method: Using `setup-java` Cache

```yaml
name: Maven Dependency Cache

on:
  push:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Setup Java with Maven Cache
        uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: 17
          cache: maven

      - name: Build Project
        run: mvn clean install
```

## Explanation

```yaml
cache: maven
```

This automatically caches Maven dependencies from:

```text
~/.m2/repository
```

First run downloads dependencies and creates cache.
Next run restores dependencies from cache and runs faster.

## Manual Method: Using `actions/cache`

```yaml
name: Manual Maven Cache

on: push

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Cache Maven Dependencies
        uses: actions/cache@v4
        with:
          path: ~/.m2/repository
          key: maven-${{ runner.os }}-${{ hashFiles('**/pom.xml') }}
          restore-keys: |
            maven-${{ runner.os }}-

      - name: Setup Java
        uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: 17

      - name: Build with Maven
        run: mvn clean install
```

## Important Parts

```yaml
path: ~/.m2/repository
```

This is Maven dependency folder.

```yaml
key: maven-${{ runner.os }}-${{ hashFiles('**/pom.xml') }}
```

Cache key changes when `pom.xml` changes.

```yaml
restore-keys:
  maven-${{ runner.os }}-
```

Fallback cache if exact cache key is not found.

## Real-Time Example

A Java Spring Boot project has many dependencies.

Without cache:

```text
Build time: 8 minutes
```

With Maven cache:

```text
Build time: 2–3 minutes
```

## Best Practices

Use `cache: maven` for simple Maven projects.
Use `actions/cache` for custom cache control.
Use `pom.xml` in cache key.
Use OS-based cache keys.
Do not cache secrets.
Do not cache final `.jar` files as dependencies.

## Simple Definition

**Maven Dependency Caching in GitHub Actions means storing Maven downloaded dependencies and reusing them in future workflow runs to make Java builds faster.**

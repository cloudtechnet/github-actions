## 61. Introduction to Workflow Caching

**Workflow Caching** means saving downloaded dependencies or build files so future workflow runs become faster.

Example:

Instead of downloading Maven dependencies every time, GitHub Actions can save them in cache and reuse them in the next run.

## Why use caching?

Caching helps to:

✅ Reduce build time
✅ Avoid downloading same dependencies again
✅ Save network usage
✅ Improve CI/CD speed
✅ Make workflows more efficient

## Common things we cache

Java Maven dependencies:

```text
~/.m2/repository
```

Node.js dependencies:

```text
~/.npm
```

Python dependencies:

```text
~/.cache/pip
```

Gradle dependencies:

```text
~/.gradle/caches
```

## Example: Maven Cache Workflow

```yaml
name: Maven Build with Cache

on:
  push:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Setup Java
        uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: 17
          cache: maven

      - name: Build Project
        run: mvn clean install
```

## Explanation

```yaml
cache: maven
```

This enables Maven dependency caching.

First run:

```text
Dependencies are downloaded and saved in cache.
```

Next run:

```text
Dependencies are restored from cache, so build is faster.
```

## Example using `actions/cache`

```yaml
name: Node Cache Example

on: push

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Cache npm dependencies
        uses: actions/cache@v4
        with:
          path: ~/.npm
          key: npm-cache-${{ hashFiles('package-lock.json') }}
          restore-keys: |
            npm-cache-

      - name: Install Dependencies
        run: npm install

      - name: Run Tests
        run: npm test
```

## Important Parts

```yaml
path: ~/.npm
```

Location to cache.

```yaml
key: npm-cache-${{ hashFiles('package-lock.json') }}
```

Unique cache key based on dependency file.

```yaml
restore-keys:
  npm-cache-
```

Fallback cache if exact match is not found.

## Simple Real-Time Example

Without cache:

```text
Every run downloads dependencies again.
Build takes more time.
```

With cache:

```text
Dependencies are reused.
Build becomes faster.
```

## Best Practices

Cache dependencies, not final build output.
Use lock files in cache key.
Use different cache keys for different OS.
Do not cache secrets.
Keep cache paths correct.

## Simple Definition

**Workflow Caching in GitHub Actions means storing dependencies or build files from one workflow run and reusing them in future runs to make CI/CD faster.**

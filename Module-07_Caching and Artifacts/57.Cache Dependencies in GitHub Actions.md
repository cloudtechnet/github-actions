## 57. Cache Dependencies in GitHub Actions

**Caching dependencies** means storing downloaded project dependencies so future workflow runs can reuse them instead of downloading again.

This makes GitHub Actions workflows much faster.

## Why cache dependencies?

Without cache:

```text
Every workflow run downloads dependencies again.
```

With cache:

```text
Dependencies are restored from saved cache.
```

Benefits:

✅ Faster builds
✅ Reduced internet downloads
✅ Better CI/CD performance
✅ Saves GitHub Actions runtime minutes

## Common Dependency Caches

| Technology  | Cache Path         |
| ----------- | ------------------ |
| Maven       | `~/.m2/repository` |
| Gradle      | `~/.gradle/caches` |
| Node.js npm | `~/.npm`           |
| Python pip  | `~/.cache/pip`     |

## Method 1: Maven Dependency Cache

## Example: Java Maven Project

```yaml
name: Maven Dependency Cache

on:
  push:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Setup Java with Maven Cache
        uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: 17
          cache: maven

      - name: Build Application
        run: mvn clean install
```

## Explanation

```yaml
cache: maven
```

GitHub automatically caches:

```text
~/.m2/repository
```

### First Run

```text
Dependencies downloaded from Maven Central.
Cache created.
```

### Next Run

```text
Dependencies restored from cache.
Build becomes faster.
```

# Method 2: Using `actions/cache`

## Example: Node.js Dependency Cache

```yaml
name: Node.js Cache Example

on:
  push:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Cache npm Dependencies
        uses: actions/cache@v4
        with:
          path: ~/.npm
          key: npm-cache-${{ hashFiles('package-lock.json') }}
          restore-keys: |
            npm-cache-

      - name: Install Dependencies
        run: npm install

      - name: Run Tests
        run: npm test
```

# Important Parts

## `path`

```yaml
path: ~/.npm
```

Location where dependencies are stored.

## `key`

```yaml
key: npm-cache-${{ hashFiles('package-lock.json') }}
```

Creates unique cache key.

If dependency file changes, cache changes.

## `restore-keys`

```yaml
restore-keys: |
  npm-cache-
```

Fallback cache matching.

# Python Dependency Cache Example

```yaml
name: Python Cache Example

on: push

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Setup Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.11"

      - name: Cache pip Packages
        uses: actions/cache@v4
        with:
          path: ~/.cache/pip
          key: pip-${{ hashFiles('requirements.txt') }}

      - name: Install Dependencies
        run: pip install -r requirements.txt
```

```text
Build time = 10 minutes
```

```text
Build time = 2 minutes
```

# Cache Key Best Practice

Good example:

```yaml
key: maven-${{ runner.os }}-${{ hashFiles('pom.xml') }}
```

This creates separate cache for:

Operating system
Dependency file version

# Best Practices

Use dependency files in cache key.
Cache only dependencies.
Do not cache secrets.
Use separate cache per OS if needed.
Clean old unused dependencies regularly.

# Simple Definition

**Caching dependencies in GitHub Actions means saving downloaded libraries/packages from one workflow run and reusing them in future runs to improve CI/CD speed.**

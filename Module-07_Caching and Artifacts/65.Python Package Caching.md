## GitHub Actions: 65. Python Package Caching

**Python Package Caching** means saving downloaded Python packages so GitHub Actions can reuse them in future workflow runs.

Python packages are usually cached from:

```text
~/.cache/pip
```

## Why use Python package caching?

Without cache:

```text
Every workflow downloads packages again.
```

With cache:

```text
Packages are restored from cache.
Build/test becomes faster.
```

## Example: Python Package Cache

```yaml
name: Python Package Cache

on:
  push:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Setup Python with Cache
        uses: actions/setup-python@v5
        with:
          python-version: "3.11"
          cache: pip

      - name: Install Dependencies
        run: pip install -r requirements.txt

      - name: Run Tests
        run: pytest
```

## Explanation

```yaml
cache: pip
```

This enables pip package caching automatically.

First run:

```text
Packages are downloaded and cache is created.
```

Next run:

```text
Packages are restored from cache.
```

## Manual Cache Example

```yaml
name: Manual Python Cache

on: push

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Cache pip packages
        uses: actions/cache@v4
        with:
          path: ~/.cache/pip
          key: pip-${{ runner.os }}-${{ hashFiles('requirements.txt') }}
          restore-keys: |
            pip-${{ runner.os }}-

      - name: Setup Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.11"

      - name: Install Dependencies
        run: pip install -r requirements.txt

      - name: Run Tests
        run: pytest
```

## Best Practices

Use `cache: pip` for simple projects.
Use `requirements.txt` in cache key.
Use OS-based cache keys.
Do not cache secrets.
Use virtual environments for clean builds.

## Simple Definition

**Python Package Caching in GitHub Actions means storing pip packages and reusing them in future workflow runs to make Python CI faster.**

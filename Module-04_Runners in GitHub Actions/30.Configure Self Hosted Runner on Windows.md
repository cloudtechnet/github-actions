# 35. Configure Self-Hosted Runner on Windows

A Windows Self-Hosted Runner allows GitHub Actions workflows to execute on your own Windows machine instead of GitHub-hosted runners.

This is useful for:

* .NET applications
* Windows software builds
* PowerShell automation
* Internal enterprise deployments
* Windows-specific tools
* Persistent CI/CD environments

---

# What is a Windows Self-Hosted Runner?

A Windows self-hosted runner is:

> A Windows machine connected to GitHub Actions to execute workflow jobs.

The machine can be:

* Windows Server
* Windows 10/11 PC
* Cloud VM
* Enterprise server
* Virtual machine

---

# Workflow Architecture

```text id="jlwmd3"
Developer Pushes Code
          ↓
GitHub Actions Triggered
          ↓
Windows Self-Hosted Runner Receives Job
          ↓
Workflow Executes on Windows Machine
          ↓
Results Sent Back to GitHub
```

# Real Use Case – IIS Deployment

---

# Workflow Example

```yaml id="jlwm1g"
name: IIS Deployment

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on:
      - self-hosted
      - windows

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Build Application
        run: dotnet build

      - name: Deploy to IIS
        run: |
          Copy-Item -Path .\publish\* `
          -Destination C:\inetpub\wwwroot\MyApp `
          -Recurse -Force
```

# Scenario

Enterprise company builds and deploys ASP.NET application to internal Windows servers.

Uses self-hosted Windows runner for:

* Internal access
* IIS deployment
* Active Directory integration

---

# Workflow

```yaml id="jlwm6f"
name: Enterprise ASP.NET Deployment

on:
  push:
    branches:
      - main

jobs:
  build-and-deploy:
    runs-on:
      - self-hosted
      - windows
      - production

    steps:
      - name: Checkout Repository
        uses: actions/checkout@v4

      - name: Restore Packages
        run: dotnet restore

      - name: Build Application
        run: dotnet build --configuration Release

      - name: Publish Application
        run: dotnet publish -o publish

      - name: Deploy to IIS
        run: |
          Copy-Item `
          -Path .\publish\* `
          -Destination C:\inetpub\wwwroot\EnterpriseApp `
          -Recurse -Force
```

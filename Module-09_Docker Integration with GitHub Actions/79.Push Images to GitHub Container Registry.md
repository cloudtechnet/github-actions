# 79. Push Images to GitHub Container Registry (GHCR)

### What is GitHub Container Registry (GHCR)?

**GitHub Container Registry (GHCR)** is GitHub's built-in container registry that allows you to securely store, manage, and distribute Docker container images directly from your GitHub repositories.

Instead of using Docker Hub, you can push Docker images to **GitHub Container Registry**, making it easy to integrate with GitHub Actions and your CI/CD pipelines.

---

## Why Use GitHub Container Registry?

Without a container registry, Docker images remain only on the local machine.

❌ Without GHCR

```text
Developer Builds Docker Image
        ↓
Image Exists Only on Local Machine
        ↓
Cannot Be Shared Easily
        ↓
Deployment Becomes Difficult
```

---

✅ With GHCR

```text
Developer Pushes Code
        ↓
GitHub Actions Builds Image
        ↓
Authenticates to GHCR
        ↓
Pushes Image
        ↓
Image Available from GitHub Registry
```

GHCR securely stores Docker images and integrates seamlessly with GitHub repositories.

---

## Benefits of GHCR

- Secure image storage
- Integrated with GitHub repositories
- Supports public and private container images
- Works seamlessly with GitHub Actions
- Supports image versioning and tagging
- Fine-grained access control using GitHub permissions

---

## Prerequisites

Before pushing images to GHCR, ensure you have:

- A GitHub repository
- A Dockerfile
- GitHub Actions enabled
- `GITHUB_TOKEN` (automatically provided by GitHub)
- Packages permission enabled

---

## Understanding GHCR Image Naming

Docker Hub image:

```text
dockerhub-user/my-app:latest
```

GHCR image:

```text
ghcr.io/github-username/my-app:latest
```

Example:

```text
ghcr.io/johndoe/my-app:v1.0
```

---

## Step 1: Configure Workflow Permissions

GitHub Actions requires permission to publish packages.

```yaml
permissions:
  contents: read
  packages: write
```

---

## Step 2: Login to GitHub Container Registry

```yaml
- name: Login to GHCR
  uses: docker/login-action@v3
  with:
    registry: ghcr.io
    username: ${{ github.actor }}
    password: ${{ secrets.GITHUB_TOKEN }}
```

GitHub automatically provides the `GITHUB_TOKEN` secret for authentication.

---

## Step 3: Build Docker Image

```yaml
- name: Build Docker Image
  run: docker build -t my-app:latest .
```

---

## Step 4: Tag the Image

Before pushing, tag the image using the GHCR naming format.

```yaml
- name: Tag Image
  run: |
    docker tag my-app:latest ghcr.io/${{ github.repository_owner }}/my-app:latest
```

Example output:

```text
ghcr.io/octocat/my-app:latest
```

---

## Step 5: Push the Image

```yaml
- name: Push Docker Image
  run: |
    docker push ghcr.io/${{ github.repository_owner }}/my-app:latest
```

The image is uploaded to GitHub Container Registry.

---

## Complete Workflow

```yaml
name: Push Image to GitHub Container Registry

on:
  workflow_dispatch:

permissions:
  contents: read
  packages: write

jobs:
  push-image:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Repository
        uses: actions/checkout@v4

      - name: Login to GHCR
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Build Docker Image
        run: docker build -t my-app:latest .

      - name: Tag Docker Image
        run: |
          docker tag my-app:latest ghcr.io/${{ github.repository_owner }}/my-app:latest

      - name: Push Docker Image
        run: |
          docker push ghcr.io/${{ github.repository_owner }}/my-app:latest
```

## Understanding the Workflow

### Workflow Permissions

```yaml
permissions:
  contents: read
  packages: write
```

Allows the workflow to:

- Read repository contents
- Publish container packages to GHCR

---

### Login to GHCR

```yaml
uses: docker/login-action@v3
```

Authenticates the workflow with GitHub Container Registry.

```yaml
registry: ghcr.io
```

Specifies GitHub Container Registry.

```yaml
username: ${{ github.actor }}
```

Uses the username of the user or workflow that triggered the workflow.

```yaml
password: ${{ secrets.GITHUB_TOKEN }}
```

Uses GitHub's automatically generated authentication token.

---

### Build Docker Image

```yaml
docker build -t my-app:latest .
```

Builds the Docker image using the Dockerfile.

---

### Tag Docker Image

```yaml
docker tag my-app:latest ghcr.io/${{ github.repository_owner }}/my-app:latest
```

Associates the local image with the GitHub Container Registry repository.

General format:

```text
ghcr.io/<github-username>/<image-name>:<tag>
```

---

### Push Docker Image

```yaml
docker push ghcr.io/${{ github.repository_owner }}/my-app:latest
```

Uploads the tagged image to GHCR.

## Push a Versioned Image

Instead of only using the **latest** tag, use version tags.

```yaml
- name: Build Image
  run: docker build -t my-app:v1.0 .

- name: Tag Image
  run: |
    docker tag my-app:v1.0 ghcr.io/${{ github.repository_owner }}/my-app:v1.0

- name: Push Image
  run: |
    docker push ghcr.io/${{ github.repository_owner }}/my-app:v1.0
```

Result:

```text
ghcr.io/octocat/my-app:v1.0
```

---

## Push Multiple Tags

```yaml
- name: Build Docker Image
  run: docker build -t my-app:latest -t my-app:v1.0 .

- name: Tag Images
  run: |
    docker tag my-app:latest ghcr.io/${{ github.repository_owner }}/my-app:latest
    docker tag my-app:v1.0 ghcr.io/${{ github.repository_owner }}/my-app:v1.0

- name: Push Images
  run: |
    docker push ghcr.io/${{ github.repository_owner }}/my-app:latest
    docker push ghcr.io/${{ github.repository_owner }}/my-app:v1.0
```

## Common Errors

### Permission Denied

```text
denied: permission_denied
```

**Solution**

Ensure the workflow includes:

```yaml
permissions:
  packages: write
```

---

### Authentication Failed

```text
unauthorized
```

**Solution**

Verify that the workflow uses:

```yaml
password: ${{ secrets.GITHUB_TOKEN }}
```

and not an invalid or expired token.

---

### Repository Not Found

```text
repository does not exist
```

**Solution**

Verify the image name and repository owner:

```text
ghcr.io/<github-username>/<image-name>
```

---

## Best Practices

✅ Use the automatically generated `GITHUB_TOKEN` whenever possible

✅ Grant only the required permissions (`packages: write`)

✅ Use version tags along with `latest`

✅ Keep container images organized by repository

✅ Push only tested and verified images

❌ Never hardcode Personal Access Tokens in workflow files

❌ Never expose authentication tokens in workflow logs

❌ Never push images without proper versioning

# 77. Docker Login using Secrets

### What is Docker Login using Secrets?

Docker Login using Secrets is the process of securely authenticating to a Docker registry (such as Docker Hub or GitHub Container Registry) by using **GitHub Secrets** instead of hardcoding usernames and passwords in the workflow.

This allows GitHub Actions to securely log in and perform operations like pushing or pulling Docker images.

---

## Why Use Secrets for Docker Login?

If credentials are written directly in the workflow file, anyone with access to the repository can view them.

❌ Bad Practice

```yaml
env:
  DOCKER_USERNAME: mydockeruser
  DOCKER_PASSWORD: mypassword123
```

Sensitive information becomes exposed.

---

✅ Good Practice

```yaml
env:
  DOCKER_USERNAME: ${{ secrets.DOCKER_USERNAME }}
  DOCKER_PASSWORD: ${{ secrets.DOCKER_PASSWORD }}
```

GitHub securely encrypts and masks secret values during workflow execution.

---

## Prerequisites

Before using Docker Login with GitHub Actions, ensure you have:

- A GitHub repository
- A Docker Hub account
- Docker Hub username
- Docker Hub Personal Access Token (recommended) or password
- GitHub Secrets configured

---

## Step 1: Create GitHub Secrets

1. Open your GitHub Repository.
2. Click **Settings**.
3. Select **Secrets and variables**.
4. Click **Actions**.
5. Select **New repository secret**.

Create the following secrets:

| Secret Name | Value |
|--------------|-------|
| DOCKER_USERNAME | Your Docker Hub username |
| DOCKER_PASSWORD | Your Docker Hub Personal Access Token |

Click **Add Secret** after entering each value.

---

## Step 2: Create GitHub Actions Workflow

Create the following file:

```text
.github/workflows/docker-login.yml
```

---

## Step 3: Login to Docker Hub

```yaml
name: Docker Login using Secrets

on:
  workflow_dispatch:

jobs:
  docker-login:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Repository
        uses: actions/checkout@v4

      - name: Login to Docker Hub
        run: |
          echo "${{ secrets.DOCKER_PASSWORD }}" | \
          docker login \
          -u "${{ secrets.DOCKER_USERNAME }}" \
          --password-stdin
```


Creates a job named **docker-login**.

---

### Runner

```yaml

```

GitHub provides an Ubuntu virtual machine to execute the workflow.

---

### Checkout Repository

```yaml
- name: Checkout Repository
  uses: actions/checkout@v4
```

Downloads the repository source code onto the runner.

---

### Docker Login

```yaml
- name: Login to Docker Hub
  run: |
    echo "${{ secrets.DOCKER_PASSWORD }}" | \
    docker login \
    -u "${{ secrets.DOCKER_USERNAME }}" \
    --password-stdin
```

Here's what each command does:

```text
echo "${{ secrets.DOCKER_PASSWORD }}"
```

Retrieves the Docker password (or Personal Access Token) stored securely in GitHub Secrets.

```text
docker login
```

Authenticates to Docker Hub.

```text
-u "${{ secrets.DOCKER_USERNAME }}"
```

Supplies the Docker Hub username.

```text
--password-stdin
```

Reads the password securely from standard input instead of exposing it in the command line.

This is the recommended and secure method for Docker authentication.

## Build and Push After Login

A real-world workflow usually logs in, builds an image, and pushes it to Docker Hub.

```yaml
name: Build and Push Docker Image

on:
  workflow_dispatch:

jobs:
  docker:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Login to Docker Hub
        run: |
          echo "${{ secrets.DOCKER_PASSWORD }}" | \
          docker login \
          -u "${{ secrets.DOCKER_USERNAME }}" \
          --password-stdin

      - name: Build Docker Image
        run: docker build -t my-app:latest .

      - name: Push Docker Image
        run: docker push my-app:latest
```

## Using the Official Docker Login Action (Recommended)

Docker provides an official GitHub Action for authentication.

```yaml
- name: Login to Docker Hub
  uses: docker/login-action@v3
  with:
    username: ${{ secrets.DOCKER_USERNAME }}
    password: ${{ secrets.DOCKER_PASSWORD }}
```

This approach is cleaner and easier to maintain.

## Accessing Secrets

Secrets are accessed using:

```yaml
${{ secrets.SECRET_NAME }}
```

Examples:

```yaml
${{ secrets.DOCKER_USERNAME }}

${{ secrets.DOCKER_PASSWORD }}

${{ secrets.GITHUB_TOKEN }}
```

---

## Secret Masking

Suppose the secret value is:

```text
mypassword123
```

GitHub automatically masks it in workflow logs.

Instead of:

```text
mypassword123
```

The log displays:

```text
***
```

This helps prevent accidental exposure of sensitive information.

---

## Common Errors

### Authentication Failed

```text
unauthorized: incorrect username or password
```

**Solution**

Verify the Docker Hub username and Personal Access Token stored in GitHub Secrets.

---

### Secret Not Found

```text
Error: Secret DOCKER_USERNAME not found
```

**Solution**

Ensure the secret names exactly match those referenced in the workflow.

---

### Login Failed

```text
Error response from daemon: login attempt failed
```

**Solution**

Confirm that the Docker Hub account is active and the credentials are valid.

---

## Best Practices

✅ Store Docker credentials as GitHub Secrets

✅ Use Docker Hub Personal Access Tokens instead of account passwords

✅ Use `--password-stdin` for secure authentication

✅ Rotate credentials periodically

✅ Grant only the minimum required permissions

❌ Never hardcode usernames or passwords in workflow files

❌ Never commit credentials to your repository

❌ Never print secret values in workflow logs

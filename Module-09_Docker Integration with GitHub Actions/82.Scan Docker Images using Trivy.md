# 82. Scan Docker Images using Trivy

### What is Trivy?

**Trivy** is an open-source security scanner developed by **Aqua Security** that scans Docker images, filesystems, Git repositories, Kubernetes clusters, and Infrastructure as Code (IaC) for security vulnerabilities and misconfigurations.

In GitHub Actions, Trivy is commonly used to automatically scan Docker images before they are pushed to a container registry or deployed to production.

---

## Why Scan Docker Images?

Docker images may contain:

- Vulnerable operating system packages
- Outdated libraries
- Known security vulnerabilities (CVEs)
- Misconfigurations
- Exposed secrets

Scanning images helps identify and fix these issues before deployment.

---

❌ Without Image Scanning

```text
Developer Pushes Code
        ↓
Docker Image Built
        ↓
Image Deployed
        ↓
Security Vulnerabilities Found in Production
```

This increases security risks and may expose applications to attacks.

---

✅ With Trivy

```text
Developer Pushes Code
        ↓
GitHub Actions Starts
        ↓
Build Docker Image
        ↓
Scan with Trivy
        ↓
No Critical Issues Found
        ↓
Deploy Application
```

Security issues are detected early in the CI/CD pipeline.

---

## Benefits of Trivy

- Fast vulnerability scanning
- Detects known CVEs
- Scans operating system packages
- Scans application dependencies
- Scans secrets
- Scans Infrastructure as Code (IaC)
- Easy integration with GitHub Actions

---

## Prerequisites

Before using Trivy, ensure you have:

- A GitHub repository
- Dockerfile
- GitHub Actions enabled
- Docker image to scan

No additional credentials are required for basic image scanning.

---

## Step 1: Build Docker Image

```yaml
- name: Build Docker Image
  run: |
    docker build -t my-app:latest .
```

---

## Step 2: Scan Image Using Trivy

```yaml
- name: Scan Docker Image
  uses: aquasecurity/trivy-action@0.28.0
  with:
    image-ref: my-app:latest
    format: table
```

Trivy scans the Docker image and reports any vulnerabilities found.

---

## Complete GitHub Actions Workflow

```yaml
name: Build and Scan Docker Image

on:
  workflow_dispatch:

jobs:
  security-scan:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Repository
        uses: actions/checkout@v4

      - name: Build Docker Image
        run: |
          docker build -t my-app:latest .

      - name: Scan Docker Image
        uses: aquasecurity/trivy-action@0.28.0
        with:
          image-ref: my-app:latest
          format: table
```

---

## Understanding the Workflow

### Checkout Repository

```yaml
uses: actions/checkout@v4
```

Downloads the repository source code.

---

### Build Docker Image

```yaml
docker build -t my-app:latest .
```

Builds the Docker image that will be scanned.

---

### Scan Docker Image

```yaml
uses: aquasecurity/trivy-action@0.28.0
```

Runs the official Trivy GitHub Action.

---

### Image Reference

```yaml
image-ref: my-app:latest
```

Specifies the Docker image to scan.

---

### Output Format

```yaml
format: table
```

Displays vulnerabilities in a readable table format.

Available formats include:

- table
- json
- sarif
- template

---

## Scan Only High and Critical Vulnerabilities

```yaml
- name: Scan Docker Image
  uses: aquasecurity/trivy-action@0.28.0
  with:
    image-ref: my-app:latest
    severity: HIGH,CRITICAL
    format: table
```

Only HIGH and CRITICAL vulnerabilities are reported.

---

## Fail Workflow on Critical Vulnerabilities

To stop the workflow if serious vulnerabilities are found:

```yaml
- name: Scan Docker Image
  uses: aquasecurity/trivy-action@0.28.0
  with:
    image-ref: my-app:latest
    severity: CRITICAL
    exit-code: 1
```

If any CRITICAL vulnerability exists, GitHub Actions marks the workflow as failed.

---

## Generate SARIF Security Report

SARIF reports can be uploaded to GitHub Security.

```yaml
- name: Scan Docker Image
  uses: aquasecurity/trivy-action@0.28.0
  with:
    image-ref: my-app:latest
    format: sarif
    output: trivy-results.sarif
```

---

## Upload SARIF Report

```yaml
- name: Upload SARIF Report
  uses: github/codeql-action/upload-sarif@v3
  with:
    sarif_file: trivy-results.sarif
```

The report appears in the repository's **Security** tab.

## Common Errors

### Image Not Found

```text
unable to inspect image
```

**Solution**

Ensure the Docker image is built successfully before running Trivy.

---

### Unsupported Image

```text
failed to analyze image
```

**Solution**

Verify that the image reference is correct.

---

### Workflow Failed

```text
exit code 1
```

**Solution**

Review the Trivy report and fix or update vulnerable packages before rebuilding the image.

---

## Best Practices

✅ Scan every Docker image before pushing it to a registry

✅ Fail the workflow for CRITICAL vulnerabilities

✅ Keep base images updated

✅ Regularly update application dependencies

✅ Review vulnerability reports before deployment

✅ Upload SARIF reports to GitHub Security for centralized tracking

❌ Don't ignore CRITICAL vulnerabilities

❌ Don't deploy unscanned Docker images

❌ Don't rely only on runtime security

# 78. Push Docker Images to Docker Hub

### What is Pushing a Docker Image to Docker Hub?

Pushing a Docker image to Docker Hub means uploading a locally built Docker image to a Docker registry so that it can be stored, shared, and pulled from anywhere.

In GitHub Actions, this process is automated after successfully building the Docker image and logging in to Docker Hub.

---

## Why Push Docker Images to Docker Hub?

Without Docker Hub, Docker images remain only on the local machine where they were built.

❌ Without Docker Hub

```text
Developer Builds Docker Image
        ↓
Image Exists Only on Local Machine
        ↓
Other Developers Cannot Access It
        ↓
Deployment Becomes Difficult
```

---

✅ With Docker Hub

```text
Developer Pushes Code
        ↓
GitHub Actions Builds Docker Image
        ↓
Logs in to Docker Hub
        ↓
Pushes Docker Image
        ↓
Image Available Anywhere
```

Docker Hub acts as a central repository for storing Docker images.

---

## Prerequisites

Before pushing Docker images to Docker Hub, ensure you have:

- A GitHub repository
- A Docker Hub account
- A Dockerfile
- Docker Hub username
- GitHub Secrets configured
- A successful Docker login

---

## Step 1: Create GitHub Secrets

Navigate to:

```text
Repository
    ↓
Settings
    ↓
Secrets and variables
    ↓
Actions
```

Create the following secrets:

| Secret Name | Value |
|--------------|-------|
| DOCKER_USERNAME | Your Docker Hub username |
| DOCKER_PASSWORD | Your Docker Hub Personal Access Token |

---

## Step 2: Build Docker Image

```yaml
- name: Build Docker Image
  run: docker build -t my-app:latest .
```

This creates a Docker image named:

```text
my-app:latest
```

---

## Step 3: Tag the Docker Image

Before pushing, the image must be tagged with your Docker Hub repository name.

```yaml
- name: Tag Docker Image
  run: docker tag my-app:latest ${{ secrets.DOCKER_USERNAME }}/my-app:latest
```

Example:

```text
mydockeruser/my-app:latest
```

---

## Step 4: Push Docker Image

```yaml
- name: Push Docker Image
  run: docker push ${{ secrets.DOCKER_USERNAME }}/my-app:latest
```

GitHub uploads the Docker image to Docker Hub.

---

## Complete Workflow

```yaml
name: Push Docker Image to Docker Hub

on:
  workflow_dispatch:

jobs:
  docker:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Repository
        uses: actions/checkout@v4

      - name: Login to Docker Hub
        uses: docker/login-action@v3
        with:
          username: ${{ secrets.DOCKER_USERNAME }}
          password: ${{ secrets.DOCKER_PASSWORD }}

      - name: Build Docker Image
        run: docker build -t my-app:latest .

      - name: Tag Docker Image
        run: docker tag my-app:latest ${{ secrets.DOCKER_USERNAME }}/my-app:latest

      - name: Push Docker Image
        run: docker push ${{ secrets.DOCKER_USERNAME }}/my-app:latest
```

## Understanding the Workflow

### Checkout Repository

```yaml
- uses: actions/checkout@v4
```

Downloads the repository source code to the GitHub Actions runner.

---

### Login to Docker Hub

```yaml
uses: docker/login-action@v3
```

Authenticates with Docker Hub using GitHub Secrets.

---

### Build Docker Image

```yaml
docker build -t my-app:latest .
```

Builds the Docker image from the Dockerfile.

---

### Tag Docker Image

```yaml
docker tag my-app:latest username/my-app:latest
```

Associates the local image with the Docker Hub repository.

General syntax:

```text
docker tag SOURCE_IMAGE TARGET_IMAGE
```

Example:

```text
docker tag my-app:latest mydockeruser/my-app:latest
```

---

### Push Docker Image

```yaml
docker push username/my-app:latest
```

Uploads the tagged image to Docker Hub.

---

GitHub Actions displays output similar to:

```text

```

This confirms the image has been uploaded successfully.

## Push a Versioned Image

Instead of using only the **latest** tag, you can push a versioned image.

```yaml
- name: Build Docker Image
  run: docker build -t my-app:v1.0 .

- name: Tag Image
  run: docker tag my-app:v1.0 ${{ secrets.DOCKER_USERNAME }}/my-app:v1.0

- name: Push Image
  run: docker push ${{ secrets.DOCKER_USERNAME }}/my-app:v1.0
```

This creates:

```text
mydockeruser/my-app:v1.0
```

---

## Push Multiple Tags

You can push both the **latest** and a version tag.

```yaml
- name: Build Docker Image
  run: docker build -t my-app:latest -t my-app:v1.0 .

- name: Tag Images
  run: |
    docker tag my-app:latest ${{ secrets.DOCKER_USERNAME }}/my-app:latest
    docker tag my-app:v1.0 ${{ secrets.DOCKER_USERNAME }}/my-app:v1.0

- name: Push Images
  run: |
    docker push ${{ secrets.DOCKER_USERNAME }}/my-app:latest
    docker push ${{ secrets.DOCKER_USERNAME }}/my-app:v1.0
```

## Common Errors

### Access Denied

```text
denied: requested access to the resource is denied
```

**Solution**

Verify that:

- You are logged in to Docker Hub.
- The repository name is correct.
- You have permission to push to the repository.

---

### Authentication Failed

```text
unauthorized: authentication required
```

**Solution**

Check the Docker Hub username and Personal Access Token stored in GitHub Secrets.

---

### Repository Does Not Exist

```text
repository does not exist
```

**Solution**

Create the repository in Docker Hub or ensure the repository name is correct.

---

## Best Practices

✅ Use GitHub Secrets for Docker credentials

✅ Push versioned images instead of relying only on `latest`

✅ Use meaningful image tags (v1.0, v2.1, release-2026)

✅ Build and push images through CI/CD pipelines

✅ Verify that the Docker image builds successfully before pushing

❌ Never hardcode Docker credentials

❌ Never push untested images to production repositories

❌ Never expose secrets in workflow logs

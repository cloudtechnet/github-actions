# 24. GitHub Context and Expressions

In GitHub Actions, **Contexts** and **Expressions** are powerful features used to create dynamic and intelligent workflows.

They help workflows:

* Access runtime information
* Make decisions
* Use variables dynamically
* Control jobs and steps
* Build advanced CI/CD pipelines

---

# What is a Context?

A **Context** is an object that contains information about:

* Workflow
* GitHub repository
* Events
* Jobs
* Steps
* Secrets
* Environment variables
* Runner system

GitHub automatically provides this information during workflow execution.

---

# Simple Definition

> Contexts are collections of data available inside workflows.

---

# Example

```yaml
${{ github.repository }}
```

This accesses repository information from the `github` context.

---

# What are Expressions?

Expressions are used to:

* Access context values
* Perform comparisons
* Apply conditions
* Execute logic dynamically

Expressions use this syntax:

```yaml
${{ expression }}
```

---

# Example

```yaml
${{ github.ref == 'refs/heads/main' }}
```

Checks whether workflow runs on main branch.

---

# Why Contexts and Expressions are Important

They make workflows:

| Feature     | Benefit                     |
| ----------- | --------------------------- |
| Dynamic     | Adapt automatically         |
| Intelligent | Use conditions              |
| Reusable    | Avoid hardcoding            |
| Flexible    | Work across environments    |
| Automated   | Trigger logic automatically |

---

# Expression Syntax

Basic syntax:

```yaml
${{ }}
```

Inside this block you can:

* Access contexts
* Compare values
* Use functions
* Write conditions

---

# Example

```yaml
run: echo "${{ github.actor }}"
```

Prints username who triggered workflow.

# 7. steps Context

Access outputs from previous steps.

---

# Step Outputs

You can create outputs using:

```yaml
echo "version=1.0" >> $GITHUB_OUTPUT
```

---

# Example

```yaml
jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - id: version_step

        run: echo "version=2.0" >> $GITHUB_OUTPUT

      - name: Read Output
        run: echo "${{ steps.version_step.outputs.version }}"
```

---

# Output

```text
2.0
```

---

# How It Works

| Part              | Meaning             |
| ----------------- | ------------------- |
| `steps`           | Access step context |
| `version_step`    | Step ID             |
| `outputs.version` | Output variable     |

# 9. needs Context

Access outputs from dependent jobs.

---

# Example

```yaml
needs: build
```

Access:

```yaml
${{ needs.build.outputs.version }}
```

---

# Example Workflow

```yaml
jobs:
  build:
    runs-on: ubuntu-latest

    outputs:
      app_version: ${{ steps.setver.outputs.version }}

    steps:
      - id: setver
        run: echo "version=1.0" >> $GITHUB_OUTPUT

  deploy:
    needs: build
    runs-on: ubuntu-latest

    steps:
      - run: echo "${{ needs.build.outputs.app_version }}"
```

# Comparison Operators

| Operator | Meaning      |
| -------- | ------------ |
| `==`     | Equal        |
| `!=`     | Not equal    |
| `>`      | Greater than |
| `<`      | Less than    |
| `&&`     | AND          |
| `\|\|`   | OR           |
| `!`      | NOT          |

---

# Example

```yaml
${{ github.ref == 'refs/heads/main' }}
```

---

# Conditional Execution

Use `if:`.

---

# Example

```yaml
steps:
  - name: Deploy
    if: github.ref == 'refs/heads/main'

    run: echo "Deploying..."
```

---

# Workflow Runs Only When

Branch = `main`

---

# Multiple Conditions

```yaml
if: github.ref == 'refs/heads/main' && github.event_name == 'push'
```

---

# Example

```yaml
steps:
  - name: Production Deploy
    if: github.ref == 'refs/heads/main' && github.actor == 'admin'

    run: echo "Production Deployment"
```

# Functions in Expressions

GitHub provides built-in functions.

---

# Common Functions

| Function       | Purpose                   |
| -------------- | ------------------------- |
| `contains()`   | Check substring           |
| `startsWith()` | Check prefix              |
| `endsWith()`   | Check suffix              |
| `success()`    | Previous steps successful |
| `failure()`    | Check failure             |
| `cancelled()`  | Check cancellation        |
| `always()`     | Run always                |

---

# Example – contains()

```yaml
if: contains(github.ref, 'main')
```

---

# Example – startsWith()

```yaml
if: startsWith(github.ref, 'refs/tags/')
```

Used for release tags.

---

# Example – success()

```yaml
if: success()
```

Runs only if previous steps succeed.

---

# Example – failure()

```yaml
if: failure()
```

Used for notifications after failures.

---

# Example

```yaml
steps:
  - name: Build
    run: exit 1

  - name: Notify Failure
    if: failure()

    run: echo "Build Failed"
```

---

# Example – always()

```yaml
if: always()
```

Runs regardless of success/failure.

# Common Mistakes

---

# Mistake 1 – Missing `${{ }}`

Wrong:

```yaml
echo github.actor
```

Correct:

```yaml
echo "${{ github.actor }}"
```

---

# Mistake 2 – Using Secrets Directly in Logs

Wrong:

```yaml
echo "${{ secrets.API_KEY }}"
```

Security risk.

---

# Mistake 3 – Wrong Context Name

Wrong:

```yaml
${{ git.actor }}
```

Correct:

```yaml
${{ github.actor }}
```

# 20. Schedule Trigger Using Cron Jobs

In GitHub Actions, the `schedule` event is used to run workflows automatically at specific times using **Cron expressions**.

This is useful for:

* Daily backups
* Automated testing
* Security scans
* Cleanup jobs
* Report generation
* Nightly builds

---

# What is a Schedule Trigger?

A schedule trigger automatically starts a workflow at a fixed time interval.

GitHub uses:

```text
Cron Syntax
```

to define the schedule.

---

# Basic Syntax

```yaml id="ec12nt"
on:
  schedule:
    - cron: '0 0 * * *'
```

---

# What is Cron?

A cron expression defines:

```text
When a workflow should run
```

Cron format contains 5 fields:

```text
* * * * *
│ │ │ │ │
│ │ │ │ └── Day of Week (0-6)
│ │ │ └──── Month (1-12)
│ │ └────── Day of Month (1-31)
│ └──────── Hour (0-23)
└────────── Minute (0-59)
```

---

# Cron Field Explanation

| Field        | Range | Example      |
| ------------ | ----- | ------------ |
| Minute       | 0–59  | `30`         |
| Hour         | 0–23  | `10`         |
| Day of Month | 1–31  | `15`         |
| Month        | 1–12  | `5`          |
| Day of Week  | 0–6   | `1` = Monday |

---

# Important Note

GitHub Actions schedule time uses:

```text
UTC Time Zone
```

not local time.

---

# Basic Example

## Run Workflow Every Day at Midnight

```yaml id="6gqlt3"
name: Daily Workflow

on:
  schedule:
    - cron: '0 0 * * *'

jobs:
  daily-job:
    runs-on: ubuntu-latest

    steps:
      - name: Print Message
        run: echo "Daily workflow executed!"
```

---

# Explanation

```text
0 0 * * *
```

Means:

| Value | Meaning       |
| ----- | ------------- |
| 0     | Minute        |
| 0     | Hour          |
| *     | Every day     |
| *     | Every month   |
| *     | Every weekday |

Result:

```text
Runs every day at 12:00 AM UTC
```

# Real-Time Example – Nightly Build

```yaml id="u6h05n"
name: Nightly Build

on:
  schedule:
    - cron: '0 2 * * *'

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Run Nightly Build
        run: echo "Executing nightly build..."
```

---

# What Happens Here?

Workflow runs:

```text
Every day at 2:00 AM UTC
```

Used for:

* Night builds
* Heavy processing
* Automated testing

# Multiple Schedule Example

You can define multiple cron schedules.

```yaml id="jlwm8s"
on:
  schedule:
    - cron: '0 9 * * 1'
    - cron: '0 18 * * 5'
```

---

# Meaning

| Cron         | Description       |
| ------------ | ----------------- |
| `0 9 * * 1`  | Every Monday 9 AM |
| `0 18 * * 5` | Every Friday 6 PM |

---

# Common Cron Examples

| Cron Expression | Meaning                  |
| --------------- | ------------------------ |
| `0 * * * *`     | Every hour               |
| `*/15 * * * *`  | Every 15 minutes         |
| `0 0 * * *`     | Daily at midnight        |
| `0 9 * * 1`     | Every Monday 9 AM        |
| `0 0 1 * *`     | First day of every month |
| `0 12 * * 0`    | Every Sunday noon        |

# Combining Schedule With Other Events

You can combine `schedule` with other triggers.

```yaml id="9oc4go"
name: Combined Workflow

on:
  push:
    branches:
      - main

  schedule:
    - cron: '0 0 * * *'

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Execute Job
        run: echo "Workflow executed!"
```

---

# What Happens?

Workflow runs when:

* Code is pushed to `main`
  OR
* Daily at midnight

---

# Important Rules

## 1. Workflow File Must Exist in Default Branch

Usually:

```text
main
```

Otherwise scheduled workflows may not run.

---

## 2. Minimum Interval

GitHub allows minimum schedule frequency:

```text
Every 5 minutes
```

Example:

```yaml id="6qrz9r"
cron: '*/5 * * * *'
```

---

# 3. Delays Can Happen

Scheduled workflows may not run exactly on time during heavy GitHub load.

Small delays are normal.

# Schedule vs Push vs Workflow Dispatch

| Event               | Trigger Type |
| ------------------- | ------------ |
| `push`              | Code push    |
| `workflow_dispatch` | Manual       |
| `schedule`          | Time-based   |

# Best Practices

✅ Use UTC time carefully
✅ Avoid very frequent schedules
✅ Use schedule for maintenance tasks
✅ Combine with notifications/logging
✅ Keep scheduled jobs optimized

# Summary

The `schedule` event in GitHub Actions is used to run workflows automatically based on time using cron expressions.

It is commonly used for:

✅ Nightly builds
✅ Backups
✅ Monitoring
✅ Security scans
✅ Maintenance tasks
✅ Automated reports

Cron jobs are a powerful way to automate recurring DevOps operations in GitHub Actions.

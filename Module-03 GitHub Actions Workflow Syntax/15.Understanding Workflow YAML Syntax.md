# Understanding Workflow YAML Syntax in GitHub Actions

GitHub Actions workflows are written using **YAML syntax**.
A workflow file defines:

* When the workflow should run
* What jobs should execute
* What steps should run inside each job
* Which operating system/environment to use

Workflow files are stored inside:

```bash
.github/workflows/
```

Example:

```bash
.github/workflows/ci.yml
```

---

# 1️⃣ Basic Workflow YAML Structure

```yaml
name: My First Workflow

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Print Message
        run: echo "Hello GitHub Actions"
```

---

# 2️⃣ Workflow YAML Main Components

| Keyword   | Purpose               |
| --------- | --------------------- |
| `name`    | Workflow name         |
| `on`      | Trigger event         |
| `jobs`    | Collection of jobs    |
| `runs-on` | OS runner             |
| `steps`   | Sequence of tasks     |
| `uses`    | Use prebuilt action   |
| `run`     | Execute shell command |
| `env`     | Environment variables |

---

# 3️⃣ `name` Keyword

Defines workflow display name.

```yaml
name: CI Pipeline
```

Shown in GitHub Actions UI.

---

# 4️⃣ `on` Keyword (Triggers)

Defines when workflow executes.

## Example: Run on Push

```yaml
on: push
```

---

## Example: Run on Push to Main Branch

```yaml
on:
  push:
    branches:
      - main
```

---

## Example: Multiple Triggers

```yaml
on:
  push:
  pull_request:
  workflow_dispatch:
```

### Meaning

| Trigger             | Description       |
| ------------------- | ----------------- |
| `push`              | Runs on code push |
| `pull_request`      | Runs during PR    |
| `workflow_dispatch` | Manual trigger    |

---

# 5️⃣ `jobs` Section

A workflow contains one or more jobs.

```yaml
jobs:
  build:
```

Here:

* `build` = Job ID

---

# 6️⃣ `runs-on`

Defines runner operating system.

```yaml
runs-on: ubuntu-latest
```

Other options:

```yaml
runs-on: windows-latest
runs-on: macos-latest
```

---

# 7️⃣ `steps`

Steps are executed sequentially inside a job.

```yaml
steps:
```

Example:

```yaml
steps:
  - name: Step 1
    run: echo "Hello"

  - name: Step 2
    run: date
```

---

# 8️⃣ `name` Inside Steps

Readable step name.

```yaml
- name: Install Dependencies
```

---

# 9️⃣ `run` Command

Executes shell commands.

```yaml
run: echo "Hello World"
```

Multiple commands:

```yaml
run: |
  pwd
  ls -la
  echo "Completed"
```

---

# 🔟 `uses` Keyword

Used to call reusable GitHub Actions.

Example:

```yaml
uses: actions/checkout@v4
```

This downloads repository code into runner.

---

# 1️⃣1️⃣ `with` Keyword

Passes input parameters to actions.

Example:

```yaml
- uses: actions/setup-node@v4
  with:
    node-version: 20
```

---

# 1️⃣2️⃣ `env` Variables

Define environment variables.

## Workflow Level

```yaml
env:
  APP_NAME: MyApplication
```

Use variable:

```yaml
run: echo $APP_NAME
```

---

## Job Level

```yaml
jobs:
  build:
    env:
      ENVIRONMENT: production
```

---

## Step Level

```yaml
steps:
  - name: Print Variable
    env:
      NAME: Rajesh
    run: echo $NAME
```

---

# 1️⃣3️⃣ Complete Workflow Example

```yaml
name: NodeJS CI Pipeline

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    env:
      APP_ENV: Production

    steps:
      - name: Checkout Repository
        uses: actions/checkout@v4

      - name: Setup NodeJS
        uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Install Dependencies
        run: npm install

      - name: Run Tests
        run: npm test

      - name: Build Application
        run: npm run build

      - name: Print Environment
        run: echo "Environment is $APP_ENV"
```

# 1️⃣4️⃣ YAML Syntax Rules

## ✅ Indentation is Important

Use spaces only.

Correct:

```yaml
jobs:
  build:
    runs-on: ubuntu-latest
```

Wrong:

```yaml
jobs:
build:
runs-on: ubuntu-latest
```

---

## ✅ Use Proper Colon (`:`)

```yaml
name: CI Workflow
```

---

## ✅ Lists Use Hyphen (`-`)

```yaml
branches:
  - main
  - develop
```

# 1️⃣6️⃣ Expressions Syntax

GitHub Actions supports expressions using:

```yaml
${{ }}
```

Example:

```yaml
run: echo "${{ github.actor }}"
```

Prints username.

---

# 1️⃣7️⃣ Access GitHub Context Variables

| Variable                   | Description |
| -------------------------- | ----------- |
| `${{ github.repository }}` | Repo name   |
| `${{ github.actor }}`      | Username    |
| `${{ github.ref }}`        | Branch      |
| `${{ github.sha }}`        | Commit ID   |

Example:

```yaml
- name: Print GitHub Info
  run: |
    echo "Repository: ${{ github.repository }}"
    echo "User: ${{ github.actor }}"
```

---

# 1️⃣8️⃣ Conditional Execution

Example:

```yaml
- name: Run Only Main Branch
  if: github.ref == 'refs/heads/main'
  run: echo "Main branch detected"
```

# 2️⃣0️⃣ Job Dependency

```yaml
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - run: echo "Build Done"

  deploy:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - run: echo "Deploy Done"
```

Meaning:

* Deploy job runs only after build completes.

---

# 2️⃣1️⃣ Real-Time DevOps CI/CD Example

```yaml
name: Java Maven CI

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Setup Java
        uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: 17

      - name: Build Maven Project
        run: mvn clean install

      - name: Upload Artifact
        uses: actions/upload-artifact@v4
        with:
          name: java-build
          path: target/*.jar
```

# 2️⃣2️⃣ Common YAML Errors

| Error                   | Cause                     |
| ----------------------- | ------------------------- |
| Invalid indentation     | Wrong spacing             |
| Missing colon           | `name CI`                 |
| Tab usage               | YAML supports spaces only |
| Wrong list syntax       | Missing `-`               |
| Wrong expression syntax | Missing `${{ }}`          |

# 70. Using Secrets in GitHub Actions Workflows

### What are Secrets?

**Secrets** are secure values stored in GitHub that contain sensitive information such as:

* Passwords
* API Keys
* Access Tokens
* Database Credentials
* Cloud Credentials (AWS, Azure, GCP)

Instead of writing sensitive data directly in your workflow file, you store it as a **GitHub Secret** and use it securely inside the workflow.

---

## Why Use Secrets?

❌ Bad Practice

```yaml
env:
  AWS_ACCESS_KEY_ID: AKIA123456789
  AWS_SECRET_ACCESS_KEY: mypassword123
```

Anyone with repository access can see these values.

---

✅ Good Practice

```yaml
env:
  AWS_ACCESS_KEY_ID: ${{ secrets.AWS_ACCESS_KEY_ID }}
  AWS_SECRET_ACCESS_KEY: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
```

GitHub automatically hides secret values in logs.

---

## Step 1: Create a Secret

1. Open GitHub Repository
2. Click **Settings**
3. Click **Secrets and variables**
4. Select **Actions**
5. Click **New repository secret**

Example:

| Secret Name           | Value           |
| --------------------- | --------------- |
| AWS_ACCESS_KEY_ID     | AKIAxxxxxxxx    |
| AWS_SECRET_ACCESS_KEY | xxxxxxxxxxxx    |
| DOCKER_USERNAME       | mydockerhubuser |
| DOCKER_PASSWORD       | mypassword      |

Save the secret.

---

## Step 2: Use Secret in Workflow

```yaml
name: Using Secrets

on:
  workflow_dispatch:

jobs:
  demo:
    runs-on: ubuntu-latest

    steps:
      - name: Display Secret
        run: echo "Secret Loaded Successfully"

      - name: Use Secret
        run: echo ${{ secrets.DOCKER_USERNAME }}
```

## Real-World Example: Docker Hub Login

```yaml
name: Docker Login

on:
  workflow_dispatch:

jobs:
  docker:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Login to Docker Hub
        run: |
          echo "${{ secrets.DOCKER_PASSWORD }}" | \
          docker login -u "${{ secrets.DOCKER_USERNAME }}" --password-stdin
```

### Workflow Flow

```text
Developer Pushes Code
        ↓
GitHub Actions Starts
        ↓
Reads Secrets Securely
        ↓
Docker Login
        ↓
Build Docker Image
        ↓
Push Image to Docker Hub
```

---

## Real-World Example: AWS Deployment

```yaml
name: Deploy to AWS

on:
  workflow_dispatch:

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Configure AWS Credentials
        uses: aws-actions/configure-aws-credentials@v4
        with:
          aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
          aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
          aws-region: us-east-1

      - name: Verify AWS Access
        run: aws s3 ls
```

## Important Points

### Access Secret

```yaml
${{ secrets.SECRET_NAME }}
```

Example:

```yaml
${{ secrets.GITHUB_TOKEN }}
${{ secrets.AWS_ACCESS_KEY_ID }}
${{ secrets.DOCKER_PASSWORD }}
```

---

### Secret Values Are Hidden

If secret value is:

```text
mypassword123
```

GitHub log shows:

```text
***
```

This prevents accidental exposure.

---

## Best Practices

✅ Store all passwords as secrets

✅ Use separate secrets for Dev, QA, and Production

✅ Rotate secrets regularly

✅ Give minimum required permissions

✅ Never hardcode credentials in YAML

❌ Never commit passwords to GitHub repository

❌ Never print secrets in logs

## Simple Classroom Example

Imagine a bank locker:

* **Repository** = Bank
* **Secret** = Locker
* **Password/API Key** = Valuable Item
* **Workflow** = Authorized Employee

The employee (workflow) can use the locker contents, but cannot expose them publicly.

**In one line:** GitHub Secrets allow you to securely store sensitive information and safely use it inside GitHub Actions workflows without exposing credentials in your code.
